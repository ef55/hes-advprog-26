Anatomy of an ADT

//   ↓ Name of the defined type
enum List [T] // ← Type parameter(s)
  //   ↓ Constructor name(s)
  case Nil ()                   extends List[T]
  case Cons(h: T, t: List[T])   extends List[T]
  //       ↑ Constructor(s)' payload
Or, in plain english:
A List of Ts is either:
  1. a Nil, made out of "nothing"
  2. a Cons, made out of a T and a List[T]
and cannot be anything else.

A Tangent Before Moving On: What Makes Them "Algebraic"

Why not just call them "fancy enums"? 
enum Tree[X] {
  case Leaf(v: X)                     extends Tree[X]
  case Fork(l: Tree[X], r: Tree[X])   extends Tree[X]
}
Tree(X) = Leaf(v: X) "or" Fork(l: Tree[X] "and" r: Tree[X]) Tree(X) = L(x: X) "or" F(t1: Tree[X] "and" t2: Tree[X]) Tree(X) = (X) "or" (Tree[X] "and" Tree[X]) Tree(X) = (X) + (Tree[X] * Tree[X])
$$T(X) = X + T(X) \cdot T(X) $$
Seems like algebra to me... and more specifically: a sum of products.
That might seem like symbol pushing, but the connection is more than syntactic.

ADTs: Summary

Algebraic Datatypes (ADTs):
  • Allow to represent finite/closed hierarchies.
  • Naturally support pattern matching extremely well.
  • Readily available (Rust, Scala, Java, etc.).
  • Fairly efficient.

GADTs: Generalizing by Constraining (Redux)

enum SmartArray[T] {
  case Default(array: Array[T])     extends SmartArray[T]
  case Compact(array: CompactArray) extends SmartArray[Boolean]
}

tldr; GADTs allow some constructors to restrict the instantiation of the type parameter to a specific type.

Case in point: Compact requires SmartArray's type parameter to be Boolean.

A Refresher(?) on Red-Black Trees

Red-Black Trees are (approximately) balanced binary search trees.

They guarante a complexity of $\log(n)$ for all basic operations:

  • insertion
  • deletion
  • membership-test

Cormen, T. H., Leiserson, C. E., Rivest, R. L., & Stein, C. Introduction to algorithms. MIT press (2002).

GADTs as Lightweight Verification

The approximate balanced part is achieved by maintaining 5 invariants:
  1. Every node is either Red or Black;
  2. The root is Black;
  3. Every leaf is Black;
  4. If a node is Red, then both its children are Black;
  5. For each node, all simple paths from the node to descendant leaves contain the same number of Black nodes.
Interested in a complete implementation (in Haskell)?
Give UPenn's CIS5520 lecture notes a read.
Weirich, S. Advanced Programming - Red Black Trees (with GADTs). University of Pennsylvania (2024).

A Scala Oddity: ADTs Declare Many Types

enum Color { // Declares 3 types: Color, Red, Black
  case Red()
  case Black()
}

val _: Color = Red()
val _: Red = Red()
val _: Black = Red() // Type error
val _ = Red() // Type defaults to Color

Red ───╼<:╾┐
           ├─ Color
Black ─╼<:╾┘
<: is the Scala symbol for "subtype"

GADTs: Summary

Generalized Algebraic Datatypes (GADTs):
  • Generalize ADTs by allowing constructors to restrict the instantiation of type parameters.
  • Seen two use cases:
    • Control over data representation (Compact Boolean Arrays).
    • Lightweight verification (Red-Black Tree).
